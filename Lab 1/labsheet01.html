CITS1401 Problem Solving and Programming

Labsheet 1

Getting Started

This labsheet is a gentle introduction to the CSSE working environment and the Python programming language. Make sure you use this time to familiarise yourself with the programming environment before commencing with the technical aspects of the unit.

Labs and After Hours Access

We will use the Mac computers in lab 2.03 of the CSSE building for all lab assignments. Even though we will use the Mac OS, Python script will run the same way in Windows and Linux operating systems. You will have 24/7 access to these labs provided your security card has been enabled for access.

Once you are enrolled in a CSSE unit, your student card should automatically allow you after-hours access to the CSSE building. If you are having any difficulties accessing the lab, please see an administrative assistant in the reception area (Room 1.31).

After-hours access will commence in the second week of semester. Outside normal working hours, make sure you carry your student card with you at all times.

Check your account

Ensure your account is working correctly by logging on.

The computers in lab 2.03 will only have Mac OS, although you can use other operating systems to complete exercises at home.

At the login screen type in your Student Number and Pheme password. Usernames and passwords are case sensitive. If you have trouble logging in, try using bookings for both username and password, and the computer will bring up a web page telling you what to do.

After successfully logging in using your account, launch Safari (on Mac) or Internet Explorer (on Windows) or Firefox (either). If you are asked for a proxy enter your University account password. This is only needed for accessing websites external to the University. To do this, you will first need to upgrade your account to a Full Service Account if you haven't already done so - see http://www.its.uwa.edu.au/student/accounts. In the meantime you should be able to hit "Cancel" and close any window that is trying to access an external site.

Navigate your way (via the School's "Current Students" page) to the on-line version of this lab sheet and proceed using the on-line version.

Starting Thonny (or PyCharm) on a Mac

Launch "Finder" by clicking on the smiley face icon at the bottom of the screen. Inside the search (top right) of the Finder, type "Thonny". The icon for Thonny should now show up inside the Finder. Double click on the Thonny icon to launch.  You can also drag the Thonny icon to the bottom of the screen and dock it there. Next time, you only need to click the already visible Thonny icon.

If you have Windows on your home computer of laptop, you can follow similar (not exactly the same) actions i.e. use search to find Thonny.

Remember to search for Thonny and not Python.

help1401

The primary and the fastest way to get help in CITS1401 will be via help1401. Try it out now, by reading and sending messages.

Check help1401 often for useful information about the unit, including about the assessments.

Python and Thonny

The suggested way of using Python with the text in this course is via an "integrated program development environment" called Thonny, available from http://thonny.org. Documentation is pretty scarce, however, though I hope using Thonny will be reasonably intuitive. Try the following.

  1. Create a folder CITS1401 in your documents folder, and a folder Lab1 inside that where you will store the work that you produce today.
  2. Launch Thonny.
  3. Try entering some expressions such as the following in the lower Shell window. Python (available through the Thonny IDE) will evaluate each one and display the result, much like a calculator.
    • 8
    • 8 * 2
    • 8 ** 2
    • 8 / 6
    • 8 // 6
    • 8 % 6
    • 8 / 0
    You can probably guess what most of the operators here do: either way we will discuss them in detail later in the unit.
  4. On Thonny's File pull-down menu, select New. This creates a new (currently untitled) top window. This is where you enter code that will be saved in a file (or code that has been loaded from a file).
  5. Enter the following into the top window. These are Python functions. All symbols and indentation are essential.

    def square(x):
            #returns the square of x
            return x * x

    def fibs(x):
            #prints the first x Fibonacci numbers
            #each number is the sum of the previous two
            a,b = 0,1
            for i in range(x):
                    print("fib(", i, ") = ", a)
                    a,b = b,a+b
  6. From the File pull-down menu, select Save. This allows you to create a permanent record of those functions.
  7. Navigate to the folder you created earlier and save the new file as lab1.py.
  8. Select Run from the Run pull-down menu (or simply click the green button with the white arrow) to load the program you have just now created. If you get any error messages at this point, you haven't entered the code correctly.
  9. In the bottom, Shell window, try entering some expressions such as the following.
    • square(-8)
    • fibs(20)
    You're running Python programs already!

Other Exercises

  1. Use the print operation from above to write a Python function that displays the name of the Python course. 

  2. (Challenge)
    Write a Python function that displays the expressions and answers to Question 3 above.

  3. From page 10 of the text, copy the chaos program (printed below for your convenience) into your lab1.py 

    def main():
        print("This program illustrates a chaotic function")
        x = eval(input("Enter a number between 0 and 1: "))
        for i in range(1,10):
            x = 3.9 * x * (1 - x)
            print(x)
    main()

Run this program and try it with different input values. Select Run -> Run Module.

(a) Modify the program using 2.0 in place of 3.9 in the expression.
(b) Modify the program so that it prints 20 values instead of 10.
(c) Modify the program so that it asks the user for the number of values to print.
(d)  Modify the prograll so that it only prints the final value of x. Change it back again so it prints all values for the next task.
(e) (Challenge) Modify the program so that it asks the user for two values of x and then prints two outputs side by side as shown in Section 1.8 of the text (page 15).

What are we learning?

As an indication of what you will be able to achieve in CITS1401, have a look at the prize-winning entries for Project 2 from the last two years. (You can find those under a separate tab on the CITS1401 LMS site.) Both years the challenge was to design a board game where the computer plays agains a human. Writing a program where the computer makes legal moves and does not allow the human to make any illegal move would fetch full marks. 

However, an extra component for brilliant students was to put some intelligence into the program where the computer plays good moves and tries to win the game.

Feel free to have a look at these programs from previous years. I am sure they will appear complicated at first but in three months time, if you work hard, you will be able to write something like this!
